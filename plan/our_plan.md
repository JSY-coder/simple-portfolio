# Portfolio Website Implementation Plan: Data Analyst

## Objective
Build a professional, modern data analyst portfolio website hosted on GitHub Pages, featuring a dark theme, JetBrains Mono font, and a clean layout highlighting three key projects and contact information.

## Key Features & Design
- **Theme:** Modern dark mode aesthetic.
- **Typography:** JetBrains Mono (monospaced for a data-focused look).
- **Hero Section:** Personal branding ("Setthawut - Data Analyst").
- **Project Section:** Showcase of 3 core data projects.
- **Contact Section:** Links for professional connection.

## Implementation Steps

### Phase 1: Setup
1.  **Initialize Directory:** Create a local folder for the project.
2.  **GitHub Repo:** Create a new repository on GitHub named `your-username.github.io` (to host at the root URL).
3.  **Local Git:** Initialize `git` locally and connect to the remote repository.

### Phase 2: Core Development
1.  **`index.html`:** Structure the hero, projects, and contact sections.
2.  **`style.css`:** Apply the dark theme (CSS variables/colors) and import/set the JetBrains Mono font.
3.  **`script.js`:** Add any necessary interactivity (e.g., smooth scroll, hover animations).

### Phase 3: Content Population
1.  **Hero:** Add the "Setthawut (Data Analyst)" header.
2.  **Projects:** Include three distinct project cards (with title, description, and link to code/visualization).
3.  **Contact:** Add buttons or links for LinkedIn, GitHub, and email.

### Phase 4: Deployment
1.  **Commit/Push:** Stage and commit all files, then push to the `main` branch.
2.  **GitHub Pages Settings:** Confirm settings are pointing to `main` branch/root directory.
3.  **Verify:** Access `https://your-username.github.io/` to confirm the live site.

## Verification & Testing
- Check responsiveness (mobile vs desktop layout).
- Verify all links (LinkedIn, GitHub, project links).
- Audit dark theme color contrast for readability.
- Validate loading time and font rendering.
